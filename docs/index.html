IBM Bob AI Coding Workshop Framework

一套可長期維護、持續擴充的教學框架，用來快速建立不同產業的 IBM Bob 實作課程。 從需求分析、Prompt 設計、產生程式碼，到檢查、重構、完成專案，教的是完整的 AI Coding Workflow。

Getting Started

不論你是要參加 Workshop 的學員，還是想貢獻新產業案例的講師，都可以從這裡開始。

給學員

  1. 安裝並登入 IBM Bob。
  2. Clone 這個 repository 到本機。
  3. 到 Workshop 頁面選一個案例（目前只有 Medical）， 打開對應 workshops/<id>/ 資料夾。
  4. 依序讀 01 故事背景 → 02 商業需求 → 03 Prompt 設計， 打開 starter/<id>/ 的起始專案，跟著 Prompt 用 Bob 開發。
  5. 卡關時看 04 Bob 操作技巧；完成後對照 05 預期成果 與 solution/<id>/。
  6. 讀 06 重點說明，再試著做 07 挑戰題。
  7. 最後用 08 課後反思 的問題整理自己走過的 AI Coding Workflow。

給講師 / 貢獻者

想新增一個新產業的 Workshop（例如 Banking、Manufacturing、Retail）， 請參考 Repository 中的 framework/CONTRIBUTING.md。

核心概念：

  • 複製 framework/workshop-template/ 到 workshops/<id>/，依固定的 8 個章節填寫內容。
  • 在 starter/、solution/、prompt-library/、instructor/ 底下新增以 <id> 命名的資料夾。
  • 在 docs/assets/data/workshops.json 註冊 metadata，網站的 Workshop 列表會自動顯示， 不需要修改任何網頁。
看 Best Practices

Workshop

每個 Workshop 都採用相同的教學架構：故事背景、商業需求、Prompt 設計、Bob 操作技巧、預期成果、重點說明、挑戰題、課後反思。 點選卡片可以到 GitHub 上閱讀完整內容。

Prompt Library

收錄各 Workshop 中可重複使用的 Prompt，依產業分類。每個 Workshop 完成後，這裡會列出對應的 Prompt 分類卡片， 可以直接連到 Repository 的 prompt-library/<id>/ 資料夾閱讀完整內容。

目前 Prompt Library 尚未有內容上線。第一批 Prompt 會隨 Medical Workshop 一併發布。

Playground

規劃中的互動練習區，讓學員不需要下載程式碼，也能直接在瀏覽器嘗試 Prompt 設計練習。

Playground 尚未開放。第一版先聚焦在 Workshop 內容本身，Playground 會在後續版本加入。

Best Practices

這個 Framework 除了教學流程，也整理 IBM Bob 的進階能力最佳實務，收錄在 Repository 的 bob/ 資料夾，供所有 Workshop 共用參考。

AGENTS

如何定義專用 Agent，什麼情境適合拆分成獨立 Agent。

Rules

如何撰寫專案規範讓 Bob 遵循，例如程式碼風格與架構限制。

Skills

如何封裝可重複使用的技能／流程給 Bob 呼叫。

Commands

如何建立自訂指令加速常見操作。

MCP 與 API Integration 的最佳實務規劃中，會在後續版本加入 bob/mcp/ 與 bob/api-integration/。
